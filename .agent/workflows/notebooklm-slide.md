# NotebookLM Slide Creator Skill

소스 문서를 분석하여 발표 슬라이드의 **구조**, **스크립트**, **디자인 컨셉**을 생성합니다.

---

## 사용 시점

- 문서, 보고서, 논문 등을 발표 슬라이드로 변환할 때
- "이 문서로 슬라이드 만들어 줘" 또는 "발표 자료 작성해 줘"라고 요청받았을 때
- 콘텐츠 분석 → 스크립트 → 디자인 제안이 필요할 때

## 사용하지 않을 때

- 이미 완성된 슬라이드의 시각적 디자인만 수정할 때
- 코드 구현이나 기술 아키텍처 작업일 때

---

## 워크플로우

```
1. 소스 분석 (Analyze)
   ↓
2. 슬라이드 구조 설계 (Structure)
   ↓
3. 슬라이드별 스크립트 작성 (Script)
   ↓
4. 디자인 컨셉 제안 (Design)
   ↓
5. 최종 출력 (Output)
```

---

## Step 1: 소스 분석 (Analyze)

사용자로부터 받은 소스를 읽고, 핵심 요소를 추출합니다.

### 추출 항목

| 항목 | 설명 | 예시 |
|:---:|---|---|
| **핵심 주제** | 문서의 중심 주제 1~3개 | "AI 에이전트 아키텍처" |
| **핵심 메시지** | 전달해야 할 핵심 메시지 | "모듈화된 설계가 확장성을 높인다" |
| **대상 청중** | 발표 대상 | 초보 개발자 / 경영진 / 투자자 |
| **키 데이터** | 숫자, 통계, 비교 | "처리 속도 3배 향상" |
| **구조적 요소** | 프로세스, 비교, 계층 | 워크플로우, 전후 비교 |
| **소스 길이** | 분량과 관계없이 **10장**으로 고정 | 핵심만 압축하여 10장 구성 |

### 분석 프롬프트 템플릿

```
다음 소스를 분석하여 아래 항목을 추출해 주세요:
1. 핵심 주제 (1~3개)
2. 핵심 메시지 (청중이 기억해야 할 것)
3. 추정 대상 청중
4. 주요 데이터/수치
5. 시각화 가능한 구조적 요소 (프로세스, 비교, 계층 등)
6. 슬라이드 수: 10장 (고정)

소스:
{source_content}
```

## Step 2: 슬라이드 구조 설계 (Structure)

### 포맷 선택

사용자에게 두 가지 포맷 중 선택하도록 안내합니다:

| 포맷 | 특징 | 적합한 상황 |
|:---:|---|---|
| **Detailed Deck** | 텍스트 중심, 정보 밀도 높음 | 보고서, 이메일 공유, 자습용 |
| **Presenter Slides** | 비주얼 중심, 핵심 키워드만 | 실제 발표, 프레젠테이션 |

### 표준 슬라이드 구조 (10장 고정)

```
 1. 타이틀 슬라이드         — 제목, 부제, 발표자, 날짜
 2. 목차 / 어젠다           — 오늘 다룰 내용 미리보기
 3~9. 본문 슬라이드 (7장)   — 핵심 내용 (주제당 1~2장)
10. 요약 / 핵심 정리        — Key Takeaways 3~5개
```

### 본문 슬라이드 유형

| 유형 | 사용 시나리오 | 시각적 요소 |
|:---:|---|---|
| **키포인트** | 핵심 메시지 전달 | 큰 텍스트 + 아이콘 |
| **비교** | A vs B, 전후 대비 | 2열 레이아웃, 테이블 |
| **프로세스** | 단계별 절차 | 플로우차트, 번호 리스트 |
| **데이터** | 수치, 통계 | 차트, 인포그래픽 |
| **인용/사례** | 실제 사례, 고객 후기 | 큰 인용 부호, 이미지 |
| **다이어그램** | 시스템 구조, 관계 | 벤 다이어그램, 아키텍처 |

## Step 3: 슬라이드별 스크립트 작성 (Script)

각 슬라이드에 대해 다음 2가지를 작성합니다:

### 슬라이드 스크립트 형식

```markdown
---
### 슬라이드 {번호}: {제목}

**슬라이드 내용** (화면에 표시될 텍스트)
- 불릿 포인트 1
- 불릿 포인트 2
- 불릿 포인트 3

**시각적 제안** (이 슬라이드에 어울리는 시각 요소)
- 레이아웃: 좌측 텍스트 + 우측 이미지
- 시각 요소: 워크플로우 다이어그램
- 강조: "3배" 수치를 대형 숫자로 표현
---
```

### 스크립트 작성 규칙

| 규칙 | 설명 |
|:---:|---|
| **1 슬라이드 = 1 메시지** | 하나의 슬라이드에 하나의 핵심 포인트만 |
| **6-6-6 법칙** | 최대 6줄, 줄당 6단어, 6번째 슬라이드마다 시각적 변화 |
| **전환 문구 포함** | "다음으로 살펴볼 것은..." 등 슬라이드 간 연결 |
| **시간 배분** | 슬라이드당 1~2분, 전체 시간에 맞춰 조절 |
| **글자 수 제한** | 최종 출력 파일 전체 **6,100자 이하** (CRLF 포함) |

## Step 4: 디자인 컨셉 제안 (Design)

### 4.1 분석 기반 컨셉 매칭

소스의 **주제, 청중, 톤**에 따라 디자인 컨셉을 자동 제안합니다:

| 콘텐츠 유형 | 추천 컨셉 | 색상 팔레트 | 폰트 스타일 |
|:---:|:---:|---|---|
| 기술/개발 | **Tech Minimal** | 다크 배경 + 네온 액센트 (`#0A0A0A`, `#00D4FF`) | Mono + Sans (Fira Code + Inter) |
| 비즈니스/IR | **Corporate Clean** | 네이비 + 화이트 + 골드 (`#1A237E`, `#FFC107`) | Serif + Sans (Playfair + Roboto) |
| 교육/튜토리얼 | **Friendly Learning** | 소프트 블루 + 오렌지 (`#E3F2FD`, `#FF7043`) | Rounded Sans (Nunito, Quicksand) |
| 창의/마케팅 | **Bold Creative** | 비비드 그라디언트 (`#FF6B6B` → `#4ECDC4`) | Display (Outfit, Space Grotesk) |
| 연구/학술 | **Academic Neutral** | 그레이 + 딥 블루 (`#F5F5F5`, `#263238`) | Classic Serif (Source Serif, Lora) |
| 스타트업/피치 | **Startup Energy** | 퍼플 + 라임 (`#6C63FF`, `#B8FF57`) | Modern Sans (General Sans, Satoshi) |

### 4.2 디자인 컨셉 출력 형식

```markdown
## 🎨 디자인 컨셉: {컨셉 이름}

**컨셉 요약**: {한 줄 설명}

### 색상 팔레트
| 용도 | 색상 | HEX |
|:---:|---|---|
| 배경 (Primary) | ██ | `#xxxxxx` |
| 텍스트 (Primary) | ██ | `#xxxxxx` |
| 강조 (Accent) | ██ | `#xxxxxx` |
| 서브 강조 | ██ | `#xxxxxx` |
| 배경 (Secondary) | ██ | `#xxxxxx` |

### 타이포그래피
| 용도 | 폰트 | 크기 | 두께 |
|:---:|---|---|---|
| 제목 (H1) | {Font} | 44pt | Bold |
| 부제 (H2) | {Font} | 28pt | SemiBold |
| 본문 | {Font} | 18pt | Regular |
| 캡션/라벨 | {Font} | 14pt | Medium |

### 레이아웃 가이드
- **여백**: 상하좌우 최소 48px
- **정렬**: {좌측 정렬 / 중앙 정렬}
- **이미지 스타일**: {둥근 모서리 8px / 그림자 / 전체 블리드}
- **아이콘**: {선형(Line) / 채움(Filled) / 듀오톤}

### 비주얼 무드
- {설명 키워드 3~5개: 예) 깔끔한, 모던, 전문적, 미니멀}
- 참고 스타일: {Apple Keynote / Pitch.com / Stripe 프레젠테이션}
```

---

## Step 5: 최종 출력 (Output)

모든 단계를 하나의 Markdown 파일로 출력합니다.

### 파일명 규칙

소스 파일명에 `ppt`를 붙여 출력합니다: `{소스파일명}ppt.md`

| 소스 파일 | 출력 파일 |
|:---:|:---:|
| `Week1.md` | `Week1ppt.md` |
| `report.md` | `reportppt.md` |

### 출력 파일 구조

```markdown
# {슬라이드 제목}

## 📊 소스 분석 요약
- 핵심 주제: ...
- 핵심 메시지: ...
- 대상 청중: ...
- 포맷: Detailed Deck / Presenter Slides
- 총 슬라이드 수: 10장
- 예상 발표 시간: M분

## 🎨 디자인 컨셉
{Step 4의 디자인 컨셉 출력}

## 📝 슬라이드 스크립트

### 슬라이드 1: {타이틀}
{Step 3의 스크립트 형식}

### 슬라이드 2: {목차}
...

### 슬라이드 10: {요약}
...
```

## 사용자 프롬프트 예시

### 기본 요청
```
이 문서로 슬라이드 만들어 줘
```

### 상세 요청
```
이 문서를 기반으로 10분 발표용 Presenter Slides를 만들어 줘.
- 청중: 비개발자 경영진
- 톤: 전문적이지만 친근한
- 강조: ROI 데이터와 고객 사례
- 디자인: Corporate Clean 스타일
```

### 부분 요청
```
이 문서를 분석해서 슬라이드 구조만 잡아 줘
이 슬라이드에 어울리는 디자인 컨셉을 제안해 줘
슬라이드 3번의 시각적 제안을 다시 써 줘
```

## 품질 체크리스트

- [ ] 각 슬라이드가 **하나의 핵심 메시지**만 전달하는가?
- [ ] **6-6-6 법칙**을 준수하는가? (최대 6줄, 줄당 6단어)
- [ ] 슬라이드 간 **전환 문구**가 포함되어 있는가?
- [ ] **시각적 제안**이 내용의 성격과 일치하는가?
- [ ] 디자인 컨셉이 **청중과 주제**에 적합한가?
- [ ] 전체 **시간 배분**이 타겟 시간에 맞는가?
- [ ] 데이터/수치에는 **시각화 제안** (차트/인포그래픽)이 있는가?
- [ ] 최종 출력 파일이 **6,100자 이하**인가?
