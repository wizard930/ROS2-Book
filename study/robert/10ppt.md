### 슬라이드 4: 노드의 구조와 생명주기

**슬라이드 내용**
- 노드 = ROS 2의 독립 실행 단위
- 하나의 노드는 하나의 모듈 목적 담당
- 생명주기: 초기화 → 생성 → 스핀 → 종료
- 하나의 실행 파일에 여러 노드 포함 가능

**시각적 제안**
- 레이아웃: 좌측 키포인트 + 우측 생명주기 플로우차트
- 시각 요소: init → spin → shutdown 4단계 흐름도
- 강조: "단일 모듈 목적" 키워드 하이라이트

### 슬라이드 5: 세 가지 통신 방식

**슬라이드 내용**
- **토픽**: 비동기, 단방향, N:N (센서 스트리밍)
- **서비스**: 동기, 양방향, 1:N (설정 변경)
- **액션**: 비동기, 양방향, 장기 (내비게이션)
- 상황에 맞는 선택이 핵심

**시각적 제안**
- 레이아웃: 3열 비교 카드
- 시각 요소: Topic·Service·Action 각각의 통신 다이어그램
- 강조: 세 방식을 서로 다른 색상으로 구분

### 슬라이드 6: 통신 방식 선택 기준

**슬라이드 내용**
- 데이터가 지속적으로 흐르는가? → Topic
- 즉각 응답이 필요한가? → Service
- 오래 걸리는 작업인가? → Action
- 의사결정 트리로 판단

**시각적 제안**
- 레이아웃: 전체 화면 의사결정 트리
- 시각 요소: Yes/No 분기 플로우차트
- 강조: 최종 선택지(Topic/Service/Action)를 Accent로

### 슬라이드 7: 메시지 — 데이터 구조 정의

**슬라이드 내용**
- `.msg` 파일로 데이터 구조 정의
- `ros2 interface show`로 구조 확인
- 표준 메시지: Twist, LaserScan, Image, Pose
- 커스텀 메시지도 직접 정의 가능

**시각적 제안**
- 레이아웃: 좌측 메시지 구조 코드 + 우측 주요 타입 표
- 시각 요소: Twist 메시지 필드 분해 다이어그램
- 강조: `geometry_msgs/msg/Twist`를 코드 블록으로

---
### 슬라이드 8: 콜백과 Executor

**슬라이드 내용**
- 메시지 도착 → 콜백 함수 자동 실행
- Executor가 콜백 실행 순서 관리
- Single: 순차 실행 / Multi: 병렬 실행
- `rclpy.spin()` 내부에서 동작

**시각적 제안**
- 레이아웃: 상단 콜백 흐름도 + 하단 Executor 비교 표
- 시각 요소: 퍼블리셔→토픽→콜백 화살표 다이어그램
- 강조: Single vs Multi를 2열 비교로 표현

---
### 슬라이드 9: Timer와 ros2doctor

**슬라이드 내용**
- Timer: 일정 주기마다 콜백 실행 (예: 2Hz)
- 센서 데이터 발행·주기적 상태 확인에 사용
- `ros2 doctor`: 환경 자동 진단 도구
- QoS 불일치·미응답 노드 등 문제 감지

**시각적 제안**
- 레이아웃: 좌측 Timer 코드 + 우측 ros2doctor 출력
- 시각 요소: 타이머 주기 파형 그래프, 진단 체크리스트
- 강조: `create_timer(0.5)`과 `ros2 doctor` 명령어
