### 슬라이드 3: ROS 2 환경 설정

**슬라이드 내용**
- 매 터미널마다 환경 소싱 필수
- `source /opt/ros/jazzy/setup.bash`
- `ROS_DOMAIN_ID`로 도메인 분리
- `ROS_LOCALHOST_ONLY`로 로컬 통신 제한

**시각적 제안**
- 레이아웃: 좌측 터미널 코드 + 우측 환경 변수 테이블
- 시각 요소: 터미널 창 모의, 환경 변수 카드
- 강조: `source` 명령어를 Accent로 하이라이트

### 슬라이드 4: 패키지 설치 — 바이너리 vs 소스

**슬라이드 내용**
- **바이너리 설치**: `apt install`로 빠르고 간편
- **소스 빌드**: `colcon build`로 커스터마이징
- 언더레이(기본 설치) 위에 오버레이(사용자 WS)
- `rosdep`으로 의존성 자동 해결

**시각적 제안**
- 레이아웃: 2열 비교 (바이너리 vs 소스)
- 시각 요소: 언더레이→오버레이 계층 구조도
- 강조: 장단점을 ✓/✕ 아이콘으로 표현

---
### 슬라이드 5: 노드 실행 — ros2 run

**슬라이드 내용**
- `ros2 run <패키지명> <실행파일명>`
- turtlesim 실행: 거북이 시뮬레이터 등장
- `turtle_teleop_key`로 키보드 조종
- `--ros-args -p`로 파라미터 설정 가능

**시각적 제안**
- 레이아웃: 좌측 터미널 명령어 + 우측 turtlesim 스크린샷
- 시각 요소: turtlesim 실행 화면 캡처
- 강조: 명령어 구조를 색상 분리로 표현

---
### 슬라이드 6: 리매핑 — 실행 시 이름 변경

**슬라이드 내용**
- 코드 수정 없이 이름 변경 가능
- 노드: `--ros-args -r __node:=my_turtle`
- 토픽: `-r /cmd_vel:=/my_cmd_vel`
- 같은 노드를 여러 개 실행할 때 유용

**시각적 제안**
- 레이아웃: 상단 명령어 + 하단 활용 시나리오 표
- 시각 요소: 이름 변환 화살표 다이어그램
- 강조: 리매핑 대상(노드/토픽/서비스) 색상 구분

---
### 슬라이드 7: 네임스페이스 — 멀티 로봇 관리

**슬라이드 내용**
- 이름 앞에 접두사를 붙여 그룹화
- `--ros-args -r __ns:=/robot1`
- 토픽·서비스도 자동으로 네임스페이스 적용
- 멀티 로봇 환경에서 충돌 없는 운용

**시각적 제안**
- 레이아웃: 전후 비교 테이블 (적용 전/후)
- 시각 요소: 2대 로봇의 독립 토픽 구조 다이어그램
- 강조: `/robot1/`, `/robot2/` 접두사 하이라이트

---
### 슬라이드 8: 런치 파일 — ros2 launch

**슬라이드 내용**
- 여러 노드를 한 번에 실행
- Python으로 런치 파일 작성
- 파라미터·네임스페이스 일괄 설정
- `ros2 run`(단일) vs `ros2 launch`(다중)

**시각적 제안**
- 레이아웃: 좌측 런치 코드 + 우측 비교 카드
- 시각 요소: 단일 실행 → 다중 실행 전환 흐름도
- 강조: `ros2 launch`를 대형 라벨로 표현

---
### 슬라이드 9: 노드 확인 도구와 rqt

**슬라이드 내용**
- `ros2 node list/info` — 노드 상태 확인
- `ros2 topic/service/action list` — 통신 확인
- rqt: GUI 기반 노드 그래프·서비스 호출·로그
- `ros2 node info`로 토픽·서비스·액션 한눈에

**시각적 제안**
- 레이아웃: 좌측 CLI 명령어 + 우측 rqt 스크린샷
- 시각 요소: rqt_graph 실행 화면, 플러그인 목록 카드
- 강조: rqt 플러그인 이름을 Accent로 표현
