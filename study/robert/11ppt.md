### 슬라이드 4: N:N 통신 — 다대다 관계

**슬라이드 내용**
- 1:1, 1:N, N:1, N:N 모두 가능
- 하나의 토픽에 여러 퍼블리셔/서브스크라이버
- 토픽마다 하나의 메시지 타입 고정
- 서로를 직접 알 필요 없는 구조

**시각적 제안**
- 레이아웃: 상단 다중 Pub/Sub 다이어그램 + 하단 특성 표

- 강조: "N:N"을 대형 숫자로 표현

### 슬라이드 5: 퍼블리셔 구현 (Python)

**슬라이드 내용**
- `create_publisher(타입, 토픽명, QoS)`
- 타이머로 주기적 메시지 발행
- `publisher_.publish(msg)`로 전송
- 핵심: 타입 · 토픽명 · QoS 세 가지 지정

**시각적 제안**
- 레이아웃: 좌측 코드 하이라이트 + 우측 흐름 요약
- 시각 요소: Python 코드의 핵심 3줄 강조
- 강조: `create_publisher`와 `publish` 함수명

---
### 슬라이드 6: 서브스크라이버 구현 (Python)

**슬라이드 내용**
- `create_subscription(타입, 토픽명, 콜백, QoS)`
- 메시지 도착 시 콜백 함수 자동 실행
- `rclpy.spin()`으로 콜백 대기
- 퍼블리셔와 동일한 토픽명·타입 필수

**시각적 제안**
- 레이아웃: 좌측 코드 하이라이트 + 우측 콜백 흐름도
- 시각 요소: 메시지→콜백 실행 화살표 다이어그램
- 강조: `listener_callback` 함수와 콜백 패턴

### 슬라이드 7: ros2 topic 명령어

**슬라이드 내용**
- `list`: 토픽 목록 (`-t`로 타입 함께)
- `echo`: 메시지 실시간 모니터링
- `pub`: 명령줄에서 직접 발행
- `info --verbose`: QoS 프로파일 확인

**시각적 제안**
- 레이아웃: 4분할 카드 (명령어별 1칸)
- 시각 요소: 터미널 출력 모의 스크린샷
- 강조: 명령어 이름을 Accent, 출력을 코드 블록으로

### 슬라이드 8: turtlesim 토픽 실습

**슬라이드 내용**

### 슬라이드 9: rqt_graph와 QoS

**슬라이드 내용**
- rqt_graph: 노드-토픽 연결 그래프 시각화
- QoS: 메시지 전달 방식 제어 (신뢰성/이력)
- 센서→BEST_EFFORT / 명령→RELIABLE
- QoS 불일치 시 토픽 연결 실패

**시각적 제안**
- 레이아웃: 상단 rqt_graph 스크린샷 + 하단 QoS 비교 표
- 시각 요소: rqt_graph 캡처, 데이터 유형별 QoS 카드
- 강조: RELIABLE vs BEST_EFFORT 색상 구분
