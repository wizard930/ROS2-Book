### 슬라이드 4: 토픽 vs 서비스 비교

**슬라이드 내용**
- **토픽**: 비동기 스트리밍, N:N, 센서 데이터 적합
- **서비스**: 동기 요청-응답, 1:N, 설정 변경 적합
- 데이터가 계속 흐르는가? (Topic) vs 필요할 때만 부르는가? (Service)
- 로봇 시스템 설계 시 상황에 맞는 선택 필수

**시각적 제안**
- 레이아웃: 2열 비교 테이블
- 시각 요소: 스트리밍(물줄기) vs 요청(문 두드리기) 아이콘 비교
- 강조: "동기"와 "비동기" 차이점 하이라이트

### 슬라이드 5: 서비스 타입 (.srv) 정의

**슬라이드 내용**
- `.srv` 파일: 요청과 응답 구조를 정의
- `---` 구분자: 위(요청) / 아래(응답)
- `interface show`로 구조 확인 가능
- 예: `AddTwoInts` (a, b 입력 → sum 출력)

**시각적 제안**
- 레이아웃: 좌측 srv 파일 구조 + 우측 설명 카드
- 시각 요소: AddTwoInts 데이터 구조 분해 다이어그램
- 강조: `---` 구분선을 굵은 선으로 강조

### 슬라이드 6: 서비스 서버 구현 (Python)

**슬라이드 내용**
- `create_service(타입, 이름, 콜백)`
- 콜백 함수에서 `request`를 받아 처리
- 결과는 `response`에 담아 반환
- 클라이언트의 요청이 올 때까지 대기

**시각적 제안**
- 레이아웃: 좌측 코드 하이라이트 + 우측 서버 동작 흐름도
- 시각 요소: Python 서버 코드의 핵심 콜백 구조 강조
- 강조: `add_two_ints_callback` 함수 내부 로직

### 슬라이드 7: 서비스 클라이언트 구현 (Python)

**슬라이드 내용**
- `create_client(타입, 이름)`
- `wait_for_service()`: 서버 준비 확인 필수
- `call_async()`: 블로킹 방지를 위한 비동기 호출
- Future 객체로 결과 완료 여부 확인

**시각적 제안**
- 레이아웃: 좌측 코드 하이라이트 + 우측 루프 흐름도
- 시각 요소: 비동기 호출(Async) 흐름 다이어그램
- 강조: `call_async`와 `Future.done()` 체크 포인트

### 슬라이드 8: ros2 service 명령어

**슬라이드 내용**

### 슬라이드 9: turtlesim 서비스 실습

**슬라이드 내용**
- `/spawn`: 새 거북이 생성
- `/kill`: 거북이 제거
- `/set_pen`: 거북이 궤적 색상/두께 변경
- `/clear`, `/reset`: 배경 및 전체 초기화

**시각적 제안**
- 레이아웃: 좌측 turtlesim 화면 + 우측 호출 명령어
- 시각 요소: `/spawn` 호출로 거북이가 추가된 화면 캡처
- 강조: 각 서비스 이름과 매칭되는 동작 설명 카드

### 슬라이드 10: 핵심 정리 및 주의사항

**슬라이드 내용**
- 서비스 = Request/Response 기반 동기 통신
- 콜백 내 동기 호출은 **데드락** 위험 → 비동기 권장
- 모든 노드는 **파라미터 서비스**를 자동 제공
- **토픽(흐름)과 서비스(요청)의 적절한 조합이 설계의 핵심**

**시각적 제안**
- 레이아웃: 중앙 정렬, 4개 핵심 카드
- 시각 요소: 주의(경고) 아이콘 + 체크 아이콘 조합
- 강조: "데드락 주의"와 "적절한 조합" 강조
