# 1장 ROS 2 소개

## 1.1 로봇 개발을 위한 툴

**ROS(Robot Operating System)** 는 이름과 달리 운영체제가 아니라, 로봇 애플리케이션 개발을 위한 **오픈소스 미들웨어 플랫폼**이다.
개발자가 통신·시각화·센서 드라이버를 처음부터 구현하는 대신, 검증된 공통 기능을 재사용하여 **핵심 로직에만 집중**할 수 있게 해준다.

**ROS가 제공하는 주요 기능**

| 기능 | 설명 |
|---|---|
| 하드웨어 추상화 | 다양한 센서·모터·액추에이터를 공통 인터페이스로 추상화 |
| 디바이스 제어 | 모터 제어, 센서 수집 등 저수준 제어 인터페이스 제공 |
| 알고리즘 라이브러리 | 센싱, 인식, SLAM, 모션 플래닝 패키지 제공 |
| 메시지 통신 | Topic, Service, Action 기반 노드 간 통신 |
| 패키지 관리 | 기능별 패키지 단위 재사용 및 배포 |
| 개발·디버깅 도구 | RViz, rqt, Gazebo 등 시각화·시뮬레이션 도구 |

---

## 1.2 메타 운영체제

ROS는 커널을 가진 전통적인 OS가 아니라, 기존 OS 위에서 동작하는 **미들웨어 계층**이다.

```
[ User Application ]
[ ROS (Middleware) ] ← 하드웨어 추상화, 분산 통신, 패키지 관리
[ OS (Linux / Windows / macOS) ]
[ Hardware ]
```

- 프로세스 관리·파일 시스템·드라이버는 기존 OS가 담당
- ROS는 노드 간 메시지 라우팅, 네임스페이스 관리, 분산 시스템 연결을 담당

---

## 1.3 ROS의 목적

> **"로보틱스 소프트웨어 개발을 전 세계 레벨에서 공동 작업이 가능하도록 하는 환경을 구축하는 것"** 

---

## 1.4 ROS의 구성

ROS는 400여 개의 ROS 2 공통 패키지로 구성되어 있다.

**대표 패키지**

| 패키지 | 기능 |
|---|---|
| Navigation2 | 2D/3D 경로 계획 및 자율 주행 |
| MoveIt | 로봇 팔 매니퓰레이션 및 모션 플래닝 |
| Cartographer | SLAM (지도 작성 및 위치 추정) |
| Gazebo | 물리 엔진 기반 3D 시뮬레이터 |
| RViz | 센서 데이터·로봇 상태 3D 시각화 |

## 1.8 버전 선택

책이 작성된지 꽤 지나서, 현 시점(2026.03) 기준으로 작성했다.

| 상황 | 권장 버전 |
|---|---|
| 처음 시작하는 경우 | 최신 LTS 버전 (Humble 또는 Jazzy) |
| 장기 프로젝트·제품 개발 | LTS 버전 |
| 최신 기능 필요 | 최신 일반 버전 |
| ROS 1 사용 중 | `ros1_bridge`로 점진적 전환 |

> ROS 2를 처음 시작하는 경우라면 고민 없이 최신 LTS 버전으로 시작하자.
