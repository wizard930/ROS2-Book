[학습 리포트] 16장. ROS 2 파라미터

1. ROS 2 파라미터의 개념 및 특징

ROS 2 파라미터는 각 노드가 실행 중에 참조하는 설정값으로, 시스템의 유연한 운영을 가능케 하는 핵심 메커니즘입니다.

* 구조적 특징: 각 노드는 개별적인 **'파라미터 서버(Parameter Server)'**를 내장하고 있습니다. 외부의 **'파라미터 클라이언트(Parameter Client)'**가 이 서버와 통신하여 값을 변경하거나 조회하는 구조입니다.
* 서비스(Service)와의 관계: 통신 방식은 서비스와 동일한 메커니즘을 공유하지만, 특정 기능을 실행하는 RPC(Remote Procedure Call)가 목적이 아닌, 노드 내 매개변수를 **지정(Set)**하거나 가져오는(Get) 데이터 관리 자체에 초점이 맞춰져 있습니다.
* 통합적 제어 (Global Variables): 각 노드는 스스로 파라미터 클라이언트가 될 수 있어, 자기 자신의 파라미터뿐만 아니라 다른 노드의 파라미터까지 읽고 쓸 수 있습니다. 이를 통해 다양한 매개변수를 마치 **'글로벌 매개변수'**처럼 활용할 수 있습니다.
* 운영 효율성: 추가적인 소스 수정이나 컴파일 과정 없이 프로세스의 동작을 실시간으로 변화시킬 수 있으며, 노드 실행 시 미리 정의된 설정 파일을 불러와 초기화할 수 있다는 점이 큰 장점입니다.
* RCL의 역할: 모든 ROS 2 노드는 RCL(ROS Client Libraries)의 기본 기능으로 파라미터 서버를 포함하고 있으므로 별도의 구현 없이 즉시 사용 가능합니다.

2. 파라미터 목록 및 상세 정보 확인

실습을 위해 두 개의 터미널에서 각각 turtlesim_node와 turtle_teleop_key를 실행합니다.

$ ros2 run turtlesim turtlesim_node
$ ros2 run turtlesim turtle_teleop_key


2.1 파라미터 목록 확인 (ros2 param list)

실행 중인 노드들의 파라미터 서버에 접근하여 현재 사용 가능한 모든 파라미터를 노드별로 나열합니다.

$ ros2 param list
/teleop_turtle:
  scale_angular
  scale_linear
  use_sim_time
/turtlesim:
  background_b
  background_g
  background_r
  use_sim_time


* use_sim_time은 모든 노드의 공통 파라미터로, 시뮬레이션 환경에서 시간 동기화를 위해 사용됩니다.
* /teleop_turtle 노드에는 거북이의 속도 제어를 위한 scale_ 계열 파라미터가 포함되어 있습니다.

2.2 파라미터 상세 정보 (ros2 param describe)

특정 파라미터가 어떤 형태인지, 어떤 제약 조건을 가지는지 상세히 파악할 수 있습니다.

/turtlesim 노드의 background_b 상세 정보 예시:

* Type: integer (정수형)
* Description: Blue channel of the background color
* Constraints:
  * Min value: 0
  * Max value: 255
  * Step: 1

3. 실시간 파라미터 값 제어 (Get/Set)

3.1 파라미터 읽기 (ros2 param get)

현재 설정된 파라미터 값을 확인합니다. turtlesim의 기본 배경색인 'Neon Blue'(#4556FF)의 실제 데이터 값은 다음과 같습니다.

$ ros2 param get /turtlesim background_r
Integer value is: 69
$ ros2 param get /turtlesim background_g
Integer value is: 86
$ ros2 param get /turtlesim background_b
Integer value is: 255


3.2 파라미터 쓰기 (ros2 param set)

컴파일 과정 없이 배경색을 'Dark Violet'(#9400D3)으로 즉시 변경해 보겠습니다. (R: 148, G: 0, B: 211)

$ ros2 param set /turtlesim background_r 148
Set parameter successful
$ ros2 param set /turtlesim background_g 0
Set parameter successful
$ ros2 param set /turtlesim background_b 211
Set parameter successful


* 명령 성공 시 Set parameter successful 메시지가 출력되며, 시뮬레이터의 배경색이 실시간으로 변경됩니다.


--------------------------------------------------------------------------------


4. 파라미터 저장 및 불러오기 (Dump/Load)

4.1 파라미터 저장 (ros2 param dump)

현재 노드에 설정된 파라미터들을 yaml 파일로 저장할 수 있습니다. 명령어 실행 시 **현재 경로(./)**에 해당 노드 이름으로 파일이 생성됩니다.

$ ros2 param dump /turtlesim
Saving to: ./turtlesim.yaml


turtlesim.yaml 파일 내부 구조:

turtlesim:
  ros__parameters:
    background_b: 211
    background_g: 0
    background_r: 148
    use_sim_time: false


4.2 저장된 파일로 노드 초기화 (Load)

노드를 다시 실행할 때 --ros-args --params-file 옵션을 사용하면 저장된 설정값으로 즉시 초기화가 가능합니다.

$ ros2 run turtlesim turtlesim_node --ros-args --params-file ./turtlesim.yaml



--------------------------------------------------------------------------------


5. 파라미터 관리 및 삭제

5.1 파라미터 삭제 (ros2 param delete)

불필요한 파라미터를 노드에서 제거할 수 있습니다.

$ ros2 param delete /turtlesim background_b
Deleted parameter successfully


5.2 최종 상태 확인

삭제 후 ros2 param list /turtlesim을 실행하여 background_b 항목이 목록에서 제외된 것을 확인하며 실습을 마무리합니다.


--------------------------------------------------------------------------------


학습 마무리: 핵심 명령어 정리

기능 설명	명령어 형식(Usage)	주요 특징
목록 확인	ros2 param list	실행 중인 모든 노드의 파라미터 나열
상세 정보	ros2 param describe <node> <param>	타입, 설명, 제약 사항(Min/Max) 확인
값 읽기	ros2 param get <node> <param>	현재 설정된 파라미터 값 출력
값 변경	ros2 param set <node> <param> <value>	실시간으로 파라미터 값 수정
설정 저장	ros2 param dump <node>	현재 설정을 현재 경로에 yaml로 저장
파라미터 삭제	ros2 param delete <node> <param>	특정 파라미터 항목을 노드에서 삭제
